前面解决了对话记忆的问题，Chain 能记住之前的聊天内容了。但有个体验问题：每次调用 `invoke()`，用户必须等到 LLM 把整个回答生成完，才能一次性看到结果。如果回答有三五百字，等待时间可能要好几秒。

流式输出（Streaming）就是解决这个问题的：让 AI 的回答一个字一个字地实时显示出来，用户不用等全部生成完。

## 基本流式输出：llm.stream()

最简单的流式输出，直接对 LLM 对象调用 `stream()` 方法：

```python
from langchain_deepseek import ChatDeepSeek

llm = ChatDeepSeek(
    model="deepseek-v4-flash",
    api_key=DEEPSEEK_API_KEY,
    base_url=DEEPSEEK_BASE_URL,
)

print("AI 回答: ", end="", flush=True)
for chunk in llm.stream("用100字介绍Python"):
    print(chunk.content, end="", flush=True)
print()
```

输出效果（文字会逐字显示）：

```
AI 回答: Python是一种简洁易读的**解释型、面向对象**高级编程语言。它以**清晰的语法**著称，强调代码的可读性。凭借海量的**标准库和第三方库**，Python广泛应用于**数据科学**、人工智能、Web开发（如Django框架）和自动化脚本。
```

这里发生了什么：

- `stream()` 返回一个迭代器，每次 yield 一个 chunk（一小段文本）
- `chunk.content` 是当前片段的文本内容
- `end=""` 让 print 不换行，`flush=True` 让内容立即显示在终端
- 整个循环结束后，所有 chunk 拼起来就是完整的回答

对比之前用的 `invoke()`：

- `invoke("用100字介绍Python")` 返回一个完整的 AIMessage，等待全部生成完才返回
- `stream("用100字介绍Python")` 返回迭代器，边生成边返回片段

## Chain 流式输出：chain.stream()

前面我们用 LCEL 拼的 Chain 也能流式输出。加一个 `StrOutputParser` 把 AIMessage 转成纯字符串：

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个 Python 专家，用中文回答，简洁明了。"),
    ("human", "{input}"),
])

chain = prompt | llm | StrOutputParser()

print("AI 回答: ", end="", flush=True)
for chunk in chain.stream({"input": "用3句话解释什么是装饰器"}):
    print(chunk, end="", flush=True)
print()
```

这里 `chain.stream()` 会把整个 Chain 的每一步都流式处理：prompt 填充完之后，LLM 流式生成，StrOutputParser 流式解析。最终 for 循环拿到的每个 chunk 是纯文本字符串（不再是 AIMessage 对象），可以直接 print。

LCEL 的流式透传是自动的——只要 Chain 中有组件支持流式（LLM 通常都支持），`stream()` 就能从末端拿到流式输出，中间的 parser、prompt 等组件会自动透传。

## 异步流式输出：chain.astream()

在 Web 服务中，通常用异步方式处理请求。LangChain 提供了异步版本的流式输出：

```python
import asyncio

async def async_stream_demo():
    prompt = ChatPromptTemplate.from_messages([
        ("system", "你是一个编程助手，用中文简洁回答。"),
        ("human", "{input}"),
    ])
    chain = prompt | llm | StrOutputParser()

    print("AI 回答: ", end="", flush=True)
    async for chunk in chain.astream({"input": "列举3个Python的Web框架"}):
        print(chunk, end="", flush=True)
    print()

# 运行异步函数
asyncio.run(async_stream_demo())
```

`astream()` 是 `stream()` 的异步版本，用 `async for` 替代 `for`，用法完全一样。在 FastAPI 等异步框架中，用 `astream()` 可以把流式输出通过 SSE（Server-Sent Events，服务器推送事件）实时发送给浏览器。

三种调用方式的对比：

| 方法 | 类型 | 返回值 | 适用场景 |
|------|------|--------|----------|
| `invoke()` | 同步 | 完整结果 | 批量处理、不需要实时显示 |
| `stream()` | 同步 | 迭代器（同步） | 命令行工具、脚本 |
| `astream()` | 异步 | 异步迭代器 | Web 服务、FastAPI、异步应用 |

| 方法 | 类型 | 返回值 | 适用场景 |
|------|------|--------|----------|
| `ainvoke()` | 异步 | 完整结果 | 异步场景下不需要流式时使用 |

`stream()` 和 `astream()` 拿到的都是最终输出片段。如果需要知道 Chain 内部发生了什么——比如 LLM 什么时候开始生成、prompt 用了什么参数——需要用到流式事件。

## 流式事件：astream_events()

`astream_events()` 比 `astream()` 更深入一层。它不只输出最终文本，而是把 Chain 执行过程中的每个步骤都作为事件发出来：

```python
async def stream_events_demo():
    chain = prompt | llm | StrOutputParser()

    event_count = 0
    async for event in chain.astream_events(
        {"input": "Python 的 GIL 是什么？一句话回答"},
        version="v2",
    ):
        event_count += 1
        kind = event["event"]

        if kind in ("on_chain_start", "on_chain_end"):
            name = event.get("name", "")
            print(f"  [{kind}] name={name}")
        elif kind == "on_llm_stream":
            content = event["data"]["chunk"].content
            if content:
                print(f"  [on_llm_stream] {repr(content)}")
        elif kind == "on_llm_end":
            print(f"  [on_llm_end] LLM 调用完成")
        elif kind == "on_parser_stream":
            chunk = event["data"]["chunk"]
            if chunk:
                print(f"  [on_parser_stream] {repr(chunk)}")

    print(f"\n总共收到 {event_count} 个事件")

asyncio.run(stream_events_demo())
```

输出（节选）：

```
[on_chain_start] name=RunnableSequence
[on_parser_stream] chunk='G'
[on_parser_stream] chunk='IL'
[on_parser_stream] chunk='（'
[on_parser_stream] chunk='全局'
[on_parser_stream] chunk='解释'
[on_parser_stream] chunk='器'
[on_parser_stream] chunk='锁'
...
[on_chain_end] name=RunnableSequence

总共收到 275 个事件
```

可以看到，一个简单的 Chain 调用内部产生了大量事件。每个事件都有一个 `event` 字段表示类型，一个 `data` 字段携带具体数据。

关键事件类型：

| 事件 | 触发时机 | data 中的关键字段 |
|------|----------|-------------------|
| `on_chain_start` | Chain 或某个组件开始执行 | `input` — 输入数据 |
| `on_chain_end` | Chain 或某个组件执行完成 | `output` — 输出数据 |
| `on_llm_start` | LLM 开始生成 | `input` — 发给 LLM 的消息 |
| `on_llm_stream` | LLM 生成一个 token 片段 | `chunk` — AIMessage 片段，`.content` 是文本 |
| `on_llm_end` | LLM 生成完成 | `output` — 完整的 AIMessage |
| `on_parser_start` | 输出解析器开始工作 | `input` — 待解析的内容 |
| `on_parser_stream` | 解析器输出一个片段 | `chunk` — 解析后的文本片段 |
| `on_parser_end` | 解析器完成 | `output` — 最终解析结果 |

`astream_events()` 的 `version="v2"` 参数是必需的，它使用新版事件格式，字段命名更规范。

实际开发中 `astream_events()` 常用于：前端需要展示 Chain 内部进度（比如"正在检索文档"、"正在生成回答"），或者需要在中间步骤做日志记录和监控。

## 流式工具调用

当 LLM 调用工具时，工具的参数也是逐字生成的。`stream()` 同样能拿到工具调用的流式片段：

```python
from langchain_core.tools import tool

@tool
def add(a: int, b: int) -> int:
    """计算两个数的和"""
    return a + b

@tool
def multiply(a: int, b: int) -> int:
    """计算两个数的乘积"""
    return a * b

llm_with_tools = llm.bind_tools([add, multiply])

accumulated_tool_calls = {}

async for chunk in llm_with_tools.astream("3加5等于多少？然后结果再乘以4"):
    if chunk.content:
        print(f"  文本片段: {chunk.content}")

    # tool_call_chunks 是工具调用的流式片段
    if chunk.tool_call_chunks:
        for tc_chunk in chunk.tool_call_chunks:
            idx = tc_chunk.get("index", 0)
            if idx not in accumulated_tool_calls:
                accumulated_tool_calls[idx] = {"name": "", "args": "", "id": ""}
            # 逐步拼接工具调用的各个字段
            if tc_chunk.get("name"):
                accumulated_tool_calls[idx]["name"] += tc_chunk["name"]
            if tc_chunk.get("args"):
                accumulated_tool_calls[idx]["args"] += tc_chunk["args"]
            if tc_chunk.get("id"):
                accumulated_tool_calls[idx]["id"] += tc_chunk["id"]

# 打印累积的工具调用结果
for idx, tc in accumulated_tool_calls.items():
    print(f"工具 [{idx}]: name={tc['name']}, args={tc['args']}, id={tc['id']}")
```

输出：

```
文本片段: 好的，我先计算3加5，然后再将结果乘以4。
工具 [0]: name=add, args={"a": 3, "b": 5}, id=call_00_qz5Q9VtJFMRkVJCCaD9O4438
```

这段代码的核心逻辑是：LLM 在流式生成过程中，工具调用的 name、args、id 都是分片到达的。用一个字典 `accumulated_tool_calls` 按工具序号（index）累积拼接，最终得到完整的工具调用信息。

`tool_call_chunks` 中的每个片段包含：

| 字段 | 说明 |
|------|------|
| `index` | 工具调用的序号（同一个回答可能调用多个工具） |
| `name` | 工具名称的片段，可能分多次到达 |
| `args` | 参数 JSON 的片段，逐步拼接成完整 JSON |
| `id` | 工具调用的唯一标识，用于后续匹配结果 |

流式工具调用在前端的典型应用场景：实时显示"正在调用 add(3, 5)..."的状态提示，而不是等工具调用完成后才一次性展示。

## 总结

这节学了 LangChain 的四种流式输出方式：

- `llm.stream()` — 最基本的流式，直接对 LLM 对象调用
- `chain.stream()` — Chain 级别的流式，LCEL 自动透传
- `chain.astream()` — 异步版本，适用于 Web 服务
- `chain.astream_events()` — 最细粒度，能拿到 Chain 内部每个步骤的事件

选择依据很简单：命令行用 `stream()`，Web 服务用 `astream()`，需要监控内部步骤用 `astream_events()`。工具调用的流式输出通过 `tool_call_chunks` 逐步拼接。
