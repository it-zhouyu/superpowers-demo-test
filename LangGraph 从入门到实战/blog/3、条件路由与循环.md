前两篇的图都是直线：节点从左到右依次执行，没有任何分支。但真实场景经常需要"如果 A 就走左边，如果 B 就走右边"。比如用户问了数学题，应该交给数学专家；问了编程问题，应该交给编程专家。

这一篇学习 `add_conditional_edges`（条件边），让图根据 State 的内容动态选择下一个节点。然后在这个基础上实现循环——一个节点执行完后回到自己，直到满足条件才退出。

## add_conditional_edges：根据 State 动态路由

`add_conditional_edges` 的用法和 `add_edge` 不同。固定边直接写 `from → to`，条件边需要一个路由函数来决定 `to` 是谁：

```python
graph.add_conditional_edges("router", route_function)
```

这行代码的意思是：当 `router` 节点执行完后，调用 `route_function(state)`，它的返回值就是下一个要执行的节点名。

来看完整示例。先定义一个路由器，把用户问题分为三类：math（数学）、code（编程）、chat（闲聊），然后交给对应的专家节点处理。

## LLM 路由器：分类用户输入

第一步：定义 State，增加一个 `classification` 字段存分类结果：

```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph.message import add_messages

class State(TypedDict):
    messages: Annotated[list, add_messages]
    classification: str
```

第二步：定义路由节点，用 LLM 做分类：

```python
from langchain_core.messages import HumanMessage, AIMessage, SystemMessage
from langchain_core.prompts import ChatPromptTemplate
from langchain_deepseek import ChatDeepSeek
import os
from dotenv import load_dotenv

load_dotenv()

llm = ChatDeepSeek(
    model=os.getenv("DEEPSEEK_MODEL", "deepseek-v4-flash"),
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url=os.getenv("DEEPSEEK_BASE_URL", "https://api.deepseek.com"),
)

def router(state: State) -> dict:
    last_message = state["messages"][-1]
    user_input = last_message.content

    prompt = ChatPromptTemplate.from_messages([
        ("system",
         "你是一个分类器。根据用户输入判断属于哪个类别。\n"
         "只回复以下三个词之一：\n"
         "- math（数学相关）\n"
         "- code（编程相关）\n"
         "- chat（其他话题）\n"
         "只回复一个词。"),
        ("human", "{input}"),
    ])

    chain = prompt | llm
    result = chain.invoke({"input": user_input})
    classification = result.content.strip().lower()

    if "math" in classification:
        classification = "math"
    elif "code" in classification:
        classification = "code"
    else:
        classification = "chat"

    return {
        "classification": classification,
        "messages": [AIMessage(content=f"[路由] 分类结果: {classification}")],
    }
```

路由节点做的事情：拿到用户输入 → 让 LLM 判断类别 → 把分类结果写入 State。关键是把分类结果存到 `classification` 字段，后面的路由函数会读这个字段。

第三步：定义三个专家节点：

```python
def math_expert(state: State) -> dict:
    user_msgs = [m for m in state["messages"] if isinstance(m, HumanMessage)]
    question = user_msgs[-1].content
    response = llm.invoke([
        SystemMessage(content="你是数学专家，简洁回答数学问题。"),
        HumanMessage(content=question),
    ])
    return {"messages": [response]}

def code_expert(state: State) -> dict:
    user_msgs = [m for m in state["messages"] if isinstance(m, HumanMessage)]
    question = user_msgs[-1].content
    response = llm.invoke([
        SystemMessage(content="你是编程专家，简洁回答编程问题。"),
        HumanMessage(content=question),
    ])
    return {"messages": [response]}

def chat_expert(state: State) -> dict:
    user_msgs = [m for m in state["messages"] if isinstance(m, HumanMessage)]
    question = user_msgs[-1].content
    response = llm.invoke([
        SystemMessage(content="你是友好的聊天助手，简洁自然地回复。"),
        HumanMessage(content=question),
    ])
    return {"messages": [response]}
```

三个节点结构相同：从 State 中取出用户消息，加上各自的 system prompt，调用 LLM。区别只在 system prompt 的内容。

第四步：定义路由函数，构建图：

```python
def route_by_classification(state: State) -> str:
    classification = state.get("classification", "chat")
    return f"{classification}_expert"

from langgraph.graph import StateGraph, START, END

graph = StateGraph(State)
graph.add_node("router", router)
graph.add_node("math_expert", math_expert)
graph.add_node("code_expert", code_expert)
graph.add_node("chat_expert", chat_expert)

graph.add_edge(START, "router")
graph.add_conditional_edges("router", route_by_classification)
graph.add_edge("math_expert", END)
graph.add_edge("code_expert", END)
graph.add_edge("chat_expert", END)

app = graph.compile()
```

**`route_by_classification` 是路由函数。** 它读取 State 中的 classification 字段，返回对应的节点名。`add_conditional_edges("router", route_by_classification)` 告诉 LangGraph：router 节点执行完后，调用这个函数决定下一步。

执行效果：

```python

```

## 条件边的映射表

`add_conditional_edges` 还支持第三个参数——映射表。刚才的路由函数直接返回节点名字符串，也可以用映射表做名字转换：

```python
graph.add_conditional_edges(
    "router",
    route_by_classification,
    {
        "math_expert": "math_expert",
        "code_expert": "code_expert",
        "chat_expert": "chat_expert",
    }
)
```

映射表的 key 是路由函数可能的返回值，value 是对应的目标节点名。如果路由函数返回的值不在映射表中，会报错。不传映射表时，LangGraph 假设返回值就是节点名。

映射表在两个场景下有用：

- 路由函数返回简写（如 `"math"`），映射到实际节点名（如 `"math_expert"`）
- 需要明确声明所有可能的路由目标，方便检查完整性

## 条件边的三种用法

条件边不只能做分叉，还可以实现循环和并行（并行用固定边即可）：

| 用法 | 路由函数逻辑 | 图结构 |
|------|------------|--------|
| 分叉路由 | 根据 State 返回不同节点名 | A → B / A → C / A → D |
| 循环 | 返回当前节点名（回到自己）或下一个节点 | A → A → A → B |
| 条件终止 | 返回 END（结束）或下一个节点 | A → B / A → END |

三种用法的区别只在路由函数的返回值。分叉返回不同的目标节点，循环返回当前节点自身，条件终止返回 END。

## 循环：计数器递增到目标值

循环是条件路由的特殊情况——路由函数在满足条件时返回当前节点名（继续循环），不满足时返回 END（退出）。

来看一个简单例子：一个节点不断给自己加 1，直到 count 达到 3：

```python
class LoopState(TypedDict):
    count: int
    messages: Annotated[list, add_messages]

def increment(state: LoopState) -> dict:
    count = state["count"] + 1
    return {
        "count": count,
        "messages": [AIMessage(content=f"当前 count = {count}")],
    }

def should_continue(state: LoopState) -> str:
    if state["count"] < 3:
        return "increment"
    return END

loop_graph = StateGraph(LoopState)
loop_graph.add_node("increment", increment)
loop_graph.add_edge(START, "increment")
loop_graph.add_conditional_edges(
    "increment",
    should_continue,
    {"increment": "increment", END: END},
)
loop_app = loop_graph.compile()
```

注意 `add_conditional_edges` 中 `"increment": "increment"` ——这就是循环的关键。`should_continue` 返回 `"increment"` 时，下一个节点还是 increment 自己，形成循环。返回 `END` 时退出循环。

执行：

```python
result = loop_app.invoke({"count": 0, "messages": []})
print(f"最终 count: {result['count']}")
for msg in result["messages"]:
    print(f"  {msg.content}")
```

输出：

```
最终 count: 3
  当前 count = 1
  当前 count = 2
  当前 count = 3
```

increment 节点执行了 3 次：count 从 0 到 1（继续）、1 到 2（继续）、2 到 3（count >= 3，返回 END）。

图的 Mermaid 描述：

```
[*] --> increment
increment --> increment: should_continue → "increment"
increment --> [*]: should_continue → END
```

一个节点、一条条件边、两种可能的路径——这就是循环的最小实现。

## Agent 循环：ReAct 模式的基础

上面的计数器循环是简化版。真实的 Agent 循环遵循一个更实用的模式：

```
agent → should_continue → tools → agent（循环）
                       → END（结束）
```

这个模式叫 ReAct（Reasoning + Acting，推理与行动），是大部分 AI Agent 的核心执行模式。思路是：

1. agent 节点调用 LLM，让模型决定下一步做什么
2. should_continue 判断模型是否需要调用工具
3. 如果需要调用工具，走 tools 节点，然后回到 agent 继续推理
4. 如果不需要调用工具（模型已经能直接回答），结束

先定义工具，并把工具绑定到 LLM：

```python
from langchain_core.tools import tool

@tool
def get_weather(city: str) -> str:
    """查询城市天气"""
    data = {"北京": "晴天 28 度", "上海": "多云 26 度"}
    return data.get(city, f"{city}天气数据暂不可用")

@tool
def calculate(expression: str) -> str:
    """计算数学表达式"""
    allowed = set("0123456789+-*/.() ")
    if not all(c in allowed for c in expression):
        return "错误: 包含不允许的字符"
    return f"{expression} = {eval(expression)}"

tools = [get_weather, calculate]
llm_with_tools = llm.bind_tools(tools)
```

`@tool` 装饰器把普通函数变成 LangChain 工具。`llm.bind_tools(tools)` 把工具列表绑定到 LLM——绑定后模型就知道有哪些工具可用，在需要时返回工具调用指令而不是直接回答。

然后定义 agent 节点和 should_continue 路由函数：

```python
class AgentState(TypedDict):
    messages: Annotated[list, add_messages]

def agent_node(state: AgentState) -> dict:
    response = llm_with_tools.invoke(state["messages"])
    return {"messages": [response]}

def should_continue(state: AgentState) -> str:
    last_message = state["messages"][-1]
    if hasattr(last_message, "tool_calls") and last_message.tool_calls:
        return "tools"
    return END
```

`should_continue` 检查最后一条消息是否包含 `tool_calls`。如果 LLM 返回了工具调用指令（表示它需要调用工具），返回 `"tools"`；否则返回 `END`。

构建图：

```python
from langgraph.prebuilt import ToolNode

agent_graph = StateGraph(AgentState)
agent_graph.add_node("agent", agent_node)

# ToolNode 是 LangGraph 预构建的工具执行节点
# 它自动读取 AIMessage 中的 tool_calls，执行对应工具，返回 ToolMessage
tool_node = ToolNode(tools)
agent_graph.add_node("tools", tool_node)

agent_graph.add_edge(START, "agent")
agent_graph.add_conditional_edges("agent", should_continue, {"tools": "tools", END: END})
agent_graph.add_edge("tools", "agent")  # 工具执行完回到 agent

agent_app = agent_graph.compile()
```

**ToolNode** 是 LangGraph 提供的预构建节点，专门执行工具调用。它自动从 State 的 messages 中读取最后一条 AIMessage 的 `tool_calls` 字段，找到对应的工具函数并执行，把结果以 `ToolMessage` 的形式追加到 messages。

执行流程：

```
用户消息 → agent（LLM 思考）
         → 返回 tool_calls → tools（执行工具）→ agent（LLM 继续思考）
         → 无 tool_calls → END（输出最终回复）
```

测试一下：

```python
result = agent_app.invoke({
    "messages": [HumanMessage(content="北京天气怎么样？")],
})

for msg in result["messages"]:
    msg_type = type(msg).__name__
    if msg_type == "HumanMessage":
        print(f"用户: {msg.content}")
    elif msg_type == "AIMessage":
        if hasattr(msg, "tool_calls") and msg.tool_calls:
            print(f"AI 调用工具: {[tc['name'] for tc in msg.tool_calls]}")
        else:
            print(f"AI 回复: {msg.content[:100]}")
    elif msg_type == "ToolMessage":
        print(f"工具结果: {msg.content}")
```

输出大致如下：

```
用户: 北京天气怎么样？
AI 调用工具: ['get_weather']
工具结果: 晴天 28 度
AI 回复: 北京今天是晴天，气温28度，天气不错！
```

agent 节点循环了两次：第一次 LLM 决定调用 get_weather 工具，第二次 LLM 拿到工具结果后生成了最终回复。

## should_continue 路由函数的模式

`should_continue` 是 Agent 循环的核心控制逻辑。不同的需求有不同的写法：

| 模式 | 判断条件 | 适用场景 |
|------|---------|---------|
| 检查 tool_calls | `last_message.tool_calls` 是否存在 | 标准 ReAct Agent |
| 检查迭代次数 | `state["iterations"] >= max_iterations` | 防止无限循环 |
| 检查状态标记 | `state.get("done", False)` | 自定义结束条件 |
| 检查消息内容 | `last_message.content` 包含特定关键词 | 基于内容的控制 |

实际项目中通常组合使用。比如同时检查 tool_calls 和迭代次数：有工具调用且未超过最大次数就继续，否则结束。

## 小结

- **add_conditional_edges** 接收一个路由函数，函数的返回值决定下一个执行的节点
- **路由函数** 接收 State，返回节点名字符串或 END
- **映射表** 是可选的第三个参数，把路由函数的返回值映射到实际节点名
- **循环** 是条件边的特殊情况：路由函数返回当前节点名就形成循环
- **ReAct 循环模式** 是 Agent 的核心：agent → should_continue → tools → agent 或 END
- **ToolNode** 是预构建的工具执行节点，自动处理 tool_calls

## 完整代码

把下面代码保存为 `step3_conditional_routing.py`，并和前面的 `config.py` 放在同一个目录下运行：

```python
"""
LangGraph 教程 - Step 3: 条件路由
=================================
演示如何使用 add_conditional_edges 实现条件分支和循环：
- 根据状态动态选择下一个节点
- 路由函数的写法
- 用条件边实现循环
"""

from typing import Annotated
from typing_extensions import TypedDict

from langchain_core.messages import HumanMessage, AIMessage, SystemMessage
from langchain_core.prompts import ChatPromptTemplate
from langchain_deepseek import ChatDeepSeek
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages

from config import DEEPSEEK_API_KEY, DEEPSEEK_MODEL, DEEPSEEK_BASE_URL

# ============================================================
# 1. 定义状态
# ============================================================
# 新增 classification 字段，用于路由决策
class State(TypedDict):
    messages: Annotated[list, add_messages]
    classification: str

# ============================================================
# 2. 初始化 LLM
# ============================================================
llm = ChatDeepSeek(
    model=DEEPSEEK_MODEL,
    api_key=DEEPSEEK_API_KEY,
    base_url=DEEPSEEK_BASE_URL,
)

# ============================================================
# 3. 定义路由节点
# ============================================================
def router(state: State) -> dict:
    """路由节点：分析用户输入，判断应该交给哪个专家处理"""
    last_message = state["messages"][-1]
    user_input = last_message.content if hasattr(last_message, "content") else str(last_message)

    prompt = ChatPromptTemplate.from_messages([
        ("system",
         "你是一个分类器。根据用户的输入，判断它属于哪个类别。\n"
         "只能回复以下三个词之一：\n"
         "- math（数学相关：计算、公式、数学概念）\n"
         "- code（编程相关：代码、技术实现、调试）\n"
         "- chat（日常聊天：其他所有话题）\n"
         "只回复一个词，不要有其他内容。"),
        ("human", "{input}"),
    ])

    chain = prompt | llm
    result = chain.invoke({"input": user_input})
    classification = result.content.strip().lower()

    # 标准化分类结果
    if "math" in classification:
        classification = "math"
    elif "code" in classification:
        classification = "code"
    else:
        classification = "chat"

    return {
        "classification": classification,
        "messages": [AIMessage(content=f"[路由] 分类结果: {classification}")],
    }

# ============================================================
# 4. 定义专家节点
# ============================================================
def math_expert(state: State) -> dict:
    """数学专家节点"""
    user_msgs = [m for m in state["messages"] if isinstance(m, HumanMessage)]
    question = user_msgs[-1].content if user_msgs else "无问题"

    response = llm.invoke([
        SystemMessage(content="你是一位数学专家。请简洁地回答数学问题，可以给出公式和简要推导。"),
        HumanMessage(content=question),
    ])
    return {"messages": [response]}

def code_expert(state: State) -> dict:
    """编程专家节点"""
    user_msgs = [m for m in state["messages"] if isinstance(m, HumanMessage)]
    question = user_msgs[-1].content if user_msgs else "无问题"

    response = llm.invoke([
        SystemMessage(content="你是一位编程专家。请简洁地回答编程问题，必要时给出代码示例。"),
        HumanMessage(content=question),
    ])
    return {"messages": [response]}

def chat_expert(state: State) -> dict:
    """聊天专家节点"""
    user_msgs = [m for m in state["messages"] if isinstance(m, HumanMessage)]
    question = user_msgs[-1].content if user_msgs else "无问题"

    response = llm.invoke([
        SystemMessage(content="你是一位友好的聊天助手。请简洁自然地回复。"),
        HumanMessage(content=question),
    ])
    return {"messages": [response]}

# ============================================================
# 5. 路由函数
# ============================================================
# add_conditional_edges 会调用这个函数，根据返回值决定下一个节点
# 返回值必须是图中已注册的节点名称
def route_by_classification(state: State) -> str:
    """根据分类结果路由到对应的专家节点"""
    classification = state.get("classification", "chat")
    return f"{classification}_expert"

# ============================================================
# 6. 构建条件路由图
# ============================================================
graph = StateGraph(State)

# 添加节点
graph.add_node("router", router)
graph.add_node("math_expert", math_expert)
graph.add_node("code_expert", code_expert)
graph.add_node("chat_expert", chat_expert)

# 添加边
graph.add_edge(START, "router")

# 条件边：根据路由函数的返回值，决定 router 之后走哪个节点
# route_by_classification 返回 "math_expert"/"code_expert"/"chat_expert"
graph.add_conditional_edges("router", route_by_classification)

# 每个专家节点执行完后都到 END
graph.add_edge("math_expert", END)
graph.add_edge("code_expert", END)
graph.add_edge("chat_expert", END)

app = graph.compile()

# ============================================================
# 7. 循环图示例（计数器循环）
# ============================================================
class LoopState(TypedDict):
    count: int
    messages: Annotated[list, add_messages]

def increment(state: LoopState) -> dict:
    """每次执行 count 加 1"""
    count = state["count"] + 1
    return {
        "count": count,
        "messages": [AIMessage(content=f"当前 count = {count}")],
    }

def should_continue(state: LoopState) -> str:
    """循环条件：count < 3 则继续循环，否则结束"""
    if state["count"] < 3:
        return "increment"
    return END

loop_graph.add_node("increment", increment)
loop_graph.add_edge(START, "increment")
loop_graph.add_conditional_edges("increment", should_continue, {"increment": "increment", END: END})
loop_app = loop_graph.compile()
```
