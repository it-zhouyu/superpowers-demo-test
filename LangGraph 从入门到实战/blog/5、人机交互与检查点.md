前四篇我们搭出了完整的 ReAct Agent：能自动推理、能调用工具、能循环执行直到任务完成。但这里有个隐患——Agent 完全自主运行，没有人把关。如果它决定发一封邮件、删一条数据库记录、或者执行一个 shell 命令，这些操作是不可逆的。Agent 调用工具的那一刻，动作就已经执行了。

人机交互（Human-in-the-Loop，简称 HITL）就是解决这个问题的：在执行关键操作之前，暂停 Agent，等人确认后再继续。LangGraph 用 interrupt 机制实现这个能力，而暂停和恢复的基础是检查点（Checkpoint）。

## 检查点：MemorySaver

先解决一个前提问题：Agent 执行到一半被暂停了，状态存在哪？下次恢复的时候怎么接着跑？

LangGraph 的做法是在图的每一步执行时自动保存状态快照。这个快照叫检查点（Checkpoint），保存当前所有状态变量的值。恢复执行时，从检查点中读取状态，接着上一步继续运行。

最简单的检查点实现是 `MemorySaver`，它把状态保存在内存中：

```python
from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()
```

使用方式是把 checkpointer 传给 `compile()`：

```python
from langgraph.prebuilt import create_react_agent

app = create_react_agent(
    model=llm,
    tools=tools,
    checkpointer=checkpointer,  # 启用检查点
)
```

加了这个参数之后，图的每次 `invoke()` 调用都会自动保存状态。这跟之前不传 checkpointer 的区别：

- 不传 checkpointer：每次 `invoke()` 都是独立的，调用完状态就没了
- 传入 checkpointer：每次 `invoke()` 的状态都被保存，后续调用可以接着上一次的状态继续

MemorySaver 是内存级别的，进程重启后数据就丢了。对于开发和测试来说完全够用，生产环境的持久化方案后续会讲。

## interrupt()：在工具里暂停执行

有了检查点的基础，现在来看 interrupt（中断）机制。核心是一个函数：`interrupt()`。

直接看例子。定义一个发邮件的工具，在真正发送之前暂停，等人确认：

```python
from langgraph.types import interrupt

@tool
def send_email(to: str, subject: str, body: str) -> str:
    """发送邮件"""
    # 暂停执行，把审批请求发给用户
    approval = interrupt(
        f"即将发送邮件:\n"
        f"  收件人: {to}\n"
        f"  主题: {subject}\n"
        f"  正文: {body}\n"
        f"是否确认发送？"
    )

    # 用户恢复时传入的值就是 approval
    if approval.get("action") == "approve":
        return f"邮件已成功发送至 {to}"
    else:
        return "邮件发送已取消"
```

这段代码的关键在 `interrupt()` 调用：

1. `interrupt(value)` 的参数 `value` 是要展示给用户的信息（字符串、字典都行）
2. 调用 `interrupt()` 后，图的执行立即暂停，状态被保存到检查点
3. 用户通过 `Command(resume=...)` 恢复执行时，`resume` 的值就是 `interrupt()` 的返回值

也就是说，`interrupt()` 在代码里看起来像一个普通函数调用，但它的行为是"暂停 + 等待 + 接收恢复值"。这个机制只在图的节点或工具中有效。

再定义一个不需要审批的工具作为对比：

```python
@tool
def search_contacts(query: str) -> str:
    """搜索联系人"""
    contacts = {
        "张三": "zhangsan@example.com",
        "李四": "lisi@example.com",
        "王五": "wangwu@example.com",
    }
    results = []
    for name, email in contacts.items():
        if query in name or query in email:
            results.append(f"{name}: {email}")
    return "\n".join(results) if results else "未找到匹配的联系人"
```

## 完整示例：发邮件前的审批

把 interrupt、checkpointer、agent 组合在一起，看完整的执行流程：

```python
from langgraph.graph import StateGraph, START, END
from langgraph.prebuilt import create_react_agent
from langgraph.checkpoint.memory import MemorySaver
from langgraph.types import interrupt, Command
from langchain_core.tools import tool
from langchain_deepseek import ChatDeepSeek

# ... 工具定义同上 ...

llm = ChatDeepSeek(
    model="deepseek-v4-flash",
    api_key="sk-xxx",
    base_url="https://api.deepseek.com",
)

checkpointer = MemorySaver()
tools = [send_email, search_contacts]

app = create_react_agent(
    model=llm,
    tools=tools,
    checkpointer=checkpointer,
)
```

注意：使用 interrupt 必须启用 checkpointer，否则状态没地方保存，暂停后也无法恢复。

第一次调用——触发中断：

```python
thread_id = "email-demo-1"
config = {"configurable": {"thread_id": thread_id}}

result = app.invoke(
    {"messages": [{"role": "user", "content": "帮我给张三发一封邮件，主题是'项目进度更新'，正文是'本周项目进展顺利，已完成核心功能开发。'"}]},
    config,
)
```

执行到这里，Agent 会调用 `send_email` 工具，工具内部触发 `interrupt()`，图暂停。`result` 返回的是暂停前的状态，不是最终结果。

关键问题是：怎么知道图被暂停了？暂停时的信息是什么？这就需要 `get_state()`。

## get_state()：检查暂停状态

```python
state = app.get_state(config)

print(f"待执行节点: {state.next}")
# 输出: ('tools',)  表示暂停在 tools 节点

if state.tasks:
    for task in state.tasks:
        if task.interrupts:
            for intr in task.interrupts:
                print(f"中断信息: {intr.value}")
# 输出:
# 即将发送邮件:
#   收件人: zhangsan@example.com
#   主题: 项目进度更新
#   正文: 本周项目进展顺利，已完成核心功能开发。
# 是否确认发送？
```

`get_state()` 返回的状态对象包含几个重要属性：

| 属性 | 说明 |
|------|------|
| `values` | 当前的状态值（字典），包含 messages 等字段 |
| `next` | 下一步要执行的节点名称（元组），暂停时不为空 |
| `tasks` | 当前待执行的任务列表，包含 interrupt 信息 |
| `created_at` | 这个检查点的创建时间 |
| `parent_config` | 父检查点的配置（用于追溯历史） |

`state.tasks` 中的每个 task 有一个 `interrupts` 列表，里面是 `Interrupt` 对象。`intr.value` 就是 `interrupt()` 调用时传入的值——也就是展示给用户的审批请求。

到这里，整个暂停-检查流程就清楚了：

1. 用户发起请求
2. Agent 决定调用 `send_email` 工具
3. 工具内部调用 `interrupt()`，把邮件信息传给调用方
4. 图暂停，状态保存到检查点
5. 调用方通过 `get_state()` 查看暂停原因
6. 根据暂停原因做决定（批准或拒绝）

## Command(resume)：恢复执行

拿到中断信息后，调用方决定批准还是拒绝，用 `Command(resume=...)` 恢复执行：

```python
from langgraph.types import Command

# 批准发送
result = app.invoke(
    Command(resume={"action": "approve"}),
    config,
)
print(f"最终结果: {result['messages'][-1].content}")
# 输出: 邮件已成功发送至 zhangsan@example.com
```

回顾 `send_email` 工具里的代码：

```python
approval = interrupt(...)

if approval.get("action") == "approve":
    return f"邮件已成功发送至 {to}"
else:
    return "邮件发送已取消"
```

`Command(resume={"action": "approve"})` 中的 `{"action": "approve"}` 就是 `interrupt()` 的返回值，赋给了 `approval` 变量。工具根据这个值决定是发送邮件还是取消操作。

拒绝的情况：

```python
# 另一个对话
thread_id = "email-demo-2"
config = {"configurable": {"thread_id": thread_id}}

app.invoke(
    {"messages": [{"role": "user", "content": "帮我给李四发邮件，主题是'会议通知'，正文是'明天下午3点开项目评审会。'"}]},
    config,
)

# 拒绝发送
result = app.invoke(
    Command(resume={"action": "reject"}),
    config,
)
print(f"最终结果: {result['messages'][-1].content}")
# 输出: 邮件发送已取消
```

同一个 `send_email` 工具，approve 就发，reject 就取消。interrupt 机制把"暂停等待决定"这件事封装得很干净——工具内部不需要知道决定是怎么做出的，只需要拿到结果。

interrupt/resume 模式的关键函数总结：

| 函数/类 | 作用 | 使用位置 |
|---------|------|----------|
| `interrupt(value)` | 暂停图执行，value 传给调用方 | 节点函数或工具内部 |
| `Command(resume=value)` | 恢复图执行，value 作为 interrupt 的返回值 | 调用方的第二次 invoke |
| `get_state(config)` | 查看当前状态（是否暂停、中断信息等） | 调用方检查状态 |
| `MemorySaver()` | 内存级别的检查点存储 | compile 时传入 |

## get_state_history()：查看检查点历史

每次 `invoke()` 都会生成一个检查点。`get_state_history()` 能看到完整的执行历史：

```python
for hist_state in app.get_state_history(config):
    msg_count = len(hist_state.values.get("messages", []))
    print(f"状态: next={hist_state.next}, 消息数={msg_count}")
```

输出类似：

```
状态: next=(), 消息数=6
状态: next=('tools',), 消息数=5
状态: next=('agent',), 消息数=4
状态: next=('agent',), 消息数=3
状态: next=('agent',), 消息数=2
状态: next=(), 消息数=1
状态: next=('agent',), 消息数=0
```

每个历史状态对应图执行过程中的一步。`next` 为空元组 `()` 表示执行完成，非空表示暂停在某个节点。消息数逐步增长，反映了 Agent 的推理过程。

这个功能在调试时特别有用——能看到 Agent 在哪一步做了什么决策、调用了什么工具、工具返回了什么结果。

## thread_id：隔离不同对话

前面所有示例都用了 `thread_id`。它的作用是隔离不同的对话：

```python
config_a = {"configurable": {"thread_id": "user-alice"}}
config_b = {"configurable": {"thread_id": "user-bob"}}

# Alice 的对话
app.invoke({"messages": [HumanMessage(content="我叫Alice")]}, config_a)

# Bob 的对话
app.invoke({"messages": [HumanMessage(content="我叫Bob")]}, config_b)

# 回到 Alice 的对话，AI 还记得她的名字
result = app.invoke({"messages": [HumanMessage(content="我叫什么？")]}, config_a)
# AI 回答: 你叫 Alice
```

`thread_id` 是 `config["configurable"]["thread_id"]` 这个路径。同一个 thread_id 下的所有 invoke 共享同一个状态链，不同的 thread_id 完全隔离。在 Web 服务中，通常每个用户会话对应一个 thread_id。

## 总结

- **检查点（Checkpoint）** 是 interrupt 机制的基础，在图的每一步自动保存状态，用 `MemorySaver` 实现内存级存储
- **interrupt(value)** 在工具或节点中调用，立即暂停图执行，value 传给调用方作为审批信息
- **Command(resume=value)** 恢复执行，value 成为 interrupt 的返回值
- **get_state()** 查看当前暂停状态，get_state_history() 查看完整检查点历史
- **thread_id** 隔离不同对话的状态，每个独立会话用不同的 thread_id

## 完整代码

把下面代码保存为 `step5_human_in_loop.py`，并和前面的 `config.py` 放在同一个目录下运行：

```python
"""
LangGraph 教程 - Step 5: Human-in-the-Loop（人工介入）

演示 interrupt + checkpoint + resume 模式：
1. 工具执行前暂停，等待人工审批
2. 审批通过后继续执行
3. 审批拒绝后取消操作
4. 用 get_state() 检查暂停状态
"""

from langgraph.graph import StateGraph, START, END
from langgraph.prebuilt import create_react_agent
from langgraph.checkpoint.memory import MemorySaver
from langgraph.types import interrupt, Command
from langchain_core.tools import tool
from langchain_deepseek import ChatDeepSeek
from config import DEEPSEEK_API_KEY, DEEPSEEK_MODEL, DEEPSEEK_BASE_URL

# ============================================================
# 定义需要人工审批的工具
# ============================================================

@tool
def send_email(to: str, subject: str, body: str) -> str:
    """发送邮件（需要人工审批）

    Args:
        to: 收件人邮箱地址
        subject: 邮件主题
        body: 邮件正文
    """
    # 调用 interrupt 暂停图执行，向用户展示审批请求
    # 用户恢复时传入的值就是 interrupt 的返回值
    approval = interrupt(
        f"即将发送邮件:\n"
        f"  收件人: {to}\n"
        f"  主题: {subject}\n"
        f"  正文: {body}\n"
        f"是否确认发送？"
    )

    # 根据用户恢复时传入的决定执行不同逻辑
    if approval.get("action") == "approve":
        return f"邮件已成功发送至 {to}"
    else:
        return "邮件发送已取消"

@tool
def search_contacts(query: str) -> str:
    """搜索联系人"""
    contacts = {
        "张三": "zhangsan@example.com",
        "李四": "lisi@example.com",
        "王五": "wangwu@example.com",
    }
    results = []
    for name, email in contacts.items():
        if query in name or query in email:
            results.append(f"{name}: {email}")
    return "\n".join(results) if results else "未找到匹配的联系人"

    model=DEEPSEEK_MODEL,
    api_key=DEEPSEEK_API_KEY,
    api_base=DEEPSEEK_BASE_URL,
)

# Demo 1: 审批通过 - 发送邮件

def demo1_approve():
    """Demo 1: 人工审批通过后发送邮件"""
    print("=" * 50)
    print("Demo 1: 人工审批 - 通过")
    print("=" * 50)

    # 创建带 checkpointer 的 agent（interrupt 必须启用 checkpointer）
    checkpointer = MemorySaver()
    tools = [send_email, search_contacts]

    app = create_react_agent(
        model=llm,
        tools=tools,
        checkpointer=checkpointer,
    )

    # 配置线程 ID（每个独立对话使用不同的 thread_id）
    thread_id = "demo-1-approve"
    config = {"configurable": {"thread_id": thread_id}}

    # 第一次调用：LLM 会决定调用 send_email 工具，工具内部触发 interrupt 暂停
    print("\n--- 第一次调用: 触发工具调用 ---")
    result = app.invoke(
        {"messages": [{"role": "user", "content": "帮我给张三发一封邮件，主题是'项目进度更新'，正文是'本周项目进展顺利，已完成核心功能开发。'"}]},
        config,
    )
    print(f"结果类型: {type(result)}")

    # 检查暂停状态
    state = app.get_state(config)
    print(f"\n--- 暂停状态检查 ---")
    print(f"下一条待执行: {state.next}")
    if state.tasks:
        for task in state.tasks:
            print(f"任务 ID: {task.id}")
            if task.interrupts:
                for intr in task.interrupts:
                    print(f"中断信息: {intr.value}")

    # 恢复执行：传入审批通过的决定
    print("\n--- 恢复执行: 审批通过 ---")
    result = app.invoke(
        Command(resume={"action": "approve"}),
        config,
    )
    print(f"最终结果: {result['messages'][-1].content}")

    # 查看完整消息历史
    print(f"\n--- 完整消息历史（共 {len(result['messages'])} 条）---")
    for i, msg in enumerate(result["messages"]):
        role = type(msg).__name__
        content = msg.content[:100] if len(msg.content) > 100 else msg.content
        print(f"  [{i}] {role}: {content}")

def demo3_inspect_state():
    """Demo 3: 使用 get_state() 检查暂停状态"""
    print("\n" + "=" * 50)
    print("Demo 3: 状态检查")
    print("=" * 50)

    checkpointer = MemorySaver()
    tools = [send_email]

    app = create_react_agent(
        model=llm,
        tools=tools,
        checkpointer=checkpointer,
    )

    thread_id = "demo-3-inspect"
    config = {"configurable": {"thread_id": thread_id}}

    # 触发 interrupt
    print("\n--- 触发中断 ---")
    app.invoke(
        {"messages": [{"role": "user", "content": "发邮件给 test@example.com，主题'测试'，正文'这是一封测试邮件'"}]},
        config,
    )

    # 详细检查状态
    state = app.get_state(config)

    print(f"\n--- 状态详情 ---")
    print(f"当前待执行节点: {state.next}")
    print(f"已执行节点: {state.created_at}")
    print(f"父状态配置: {state.parent_config}")

    # 查看消息历史
    print(f"\n--- 已有消息 ---")
    for i, msg in enumerate(state.values.get("messages", [])):
        role = type(msg).__name__
        content = msg.content[:80] if len(msg.content) > 80 else msg.content
        print(f"  [{i}] {role}: {content}")

    # 查看 interrupt 详情
    print(f"\n--- 中断信息 ---")
    if state.tasks:
        for task in state.tasks:
            print(f"  任务: {task.name} (id: {task.id})")
            if task.interrupts:
                for intr in task.interrupts:
                    print(f"  中断值: {intr.value}")
                    print(f"  中断 ID: {intr.id}")
    else:
        print("  无活跃中断")

    # 查看状态历史
    print(f"\n--- 状态历史 ---")
    history_count = 0
    for hist_state in app.get_state_history(config):
        history_count += 1
        if history_count <= 5:
            print(f"  状态 {history_count}: next={hist_state.next}, "
                  f"消息数={len(hist_state.values.get('messages', []))}")
    print(f"  共 {history_count} 个历史状态")

    # 清理：恢复并取消
    print("\n--- 恢复并取消 ---")
    result = app.invoke(
        Command(resume={"action": "reject"}),
        config,
    )
    print(f"最终结果: {result['messages'][-1].content}")
```
