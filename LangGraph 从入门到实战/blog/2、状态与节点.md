前面搭出了第一个 LangGraph 图：一个 chatbot 节点、一条直线边、一个只有 messages 的 State。实际应用中，State 通常不止一个字段。比如一个客服系统，除了对话消息，还需要跟踪当前话题、处理步骤数、用户情绪等。

这一篇解决两个问题：State 怎么定义多个字段、多个字段怎么各自用不同的合并策略；以及多个节点如何分工协作，各自读写 State 的不同部分。

## 多字段 State

前面讲到的 State 只有一个 messages：

```python
class State(TypedDict):
    messages: Annotated[list, add_messages]
```

实际应用中 State 通常包含多个字段。来看一个更完整的例子：

```python
import operator
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph.message import add_messages

class State(TypedDict):
    messages: Annotated[list, add_messages]
    context: str
    step_count: Annotated[int, operator.add]
```

三个字段，三种合并策略：

- `messages`：用 `add_messages` reducer，新消息追加到列表
- `context`：没有 reducer，新值直接覆盖旧值
- `step_count`：用 `operator.add` reducer，新值和旧值做加法（累加器）

**reducer（合并器）** 的作用是定义"当节点返回某个字段的值时，怎么和 State 中已有的值合并"。没有 reducer 的字段执行直接覆盖；有 reducer 的字段执行 reducer 指定的合并逻辑。

## Reducer 的三种合并策略

用一个具体例子看清三种策略的区别。假设 State 中 `step_count` 当前值是 `2`，`context` 当前值是 `"数学"`，`messages` 当前有两条消息：

| 字段 | 当前值 | 节点返回值 | reducer | 合并后的值 |
|------|--------|-----------|---------|-----------|
| `messages` | `[msg1, msg2]` | `[msg3]` | `add_messages` | `[msg1, msg2, msg3]`（追加） |
| `context` | `"数学"` | `"编程"` | 无 | `"编程"`（覆盖） |
| `step_count` | `2` | `1` | `operator.add` | `3`（2+1，累加） |

`add_messages` 不只是简单追加。如果两条消息的 id 相同，它会用新消息替换旧消息，而不是追加。这个特性在需要修改历史消息时很有用（比如多 Agent 场景中一个 Agent 修正另一个 Agent 的输出）。普通追加场景下不需要关心这个细节。

`operator.add` 也可以用在 list 类型上，效果是列表拼接。但大部分场景用在 int 上做计数器。

| Reducer | 适用类型 | 合并逻辑 | 典型用途 |
|---------|---------|---------|---------|
| `add_messages` | list | 追加消息（相同 id 则替换） | 对话历史 |
| `operator.add` | int / list | 加法（int）或拼接（list） | 计数器、日志收集 |
| 无 | 任意 | 直接覆盖 | 上下文、状态标记、配置 |

## 节点函数的签名和返回值

每个节点函数接收当前 State，返回一个字典。字典的 key 是要更新的 State 字段，value 是新值：

```python
def analyzer(state: State) -> dict:
    last_message = state["messages"][-1]
    user_input = last_message.content

    # 用 LLM 分析用户输入的主题
    analysis = llm.invoke([HumanMessage(content=f"请用一句话概括以下输入的主题：\n\n{user_input}")])

    return {
        "context": f"主题: {analysis.content}",
        "messages": [AIMessage(content=f"[分析] 用户在讨论: {analysis.content}")],
        "step_count": 1,
    }
```

注意返回值不是完整的 State，只包含需要更新的字段。不需要修改的字段可以不返回（比如这个节点不修改 context 以外的 str 字段时，context 会更新，不返回的字段保持原值）。

**`step_count: 1` 不是把 step_count 设为 1**，而是给 step_count 加 1。因为这个字段声明了 `operator.add` reducer，返回值 1 会和当前值做加法。如果想让计数器加 2，就返回 2。

## 多节点处理链：analyzer → responder → counter

把三个节点串起来，每个节点负责不同的任务：

```python
from langchain_core.messages import HumanMessage, AIMessage
from langchain_deepseek import ChatDeepSeek
from langgraph.graph import StateGraph, START, END
import os
from dotenv import load_dotenv

load_dotenv()

llm = ChatDeepSeek(
    model=os.getenv("DEEPSEEK_MODEL", "deepseek-v4-flash"),
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url=os.getenv("DEEPSEEK_BASE_URL", "https://api.deepseek.com"),
)

def analyzer(state: State) -> dict:
    """分析节点：提取用户输入的主题"""
    last_message = state["messages"][-1]
    user_input = last_message.content

    analysis = llm.invoke([HumanMessage(content=f"请用一句话概括以下输入的主题：\n\n{user_input}")])

    return {
        "context": f"主题: {analysis.content}",
        "messages": [AIMessage(content=f"[分析] 用户在讨论: {analysis.content}")],
        "step_count": 1,
    }

def responder(state: State) -> dict:
    """回复节点：根据上下文回复用户"""
    context_info = state.get("context", "无上下文")
    user_msgs = [m for m in state["messages"] if isinstance(m, HumanMessage)]
    last_user_msg = user_msgs[-1]

    response = llm.invoke([
        HumanMessage(content=f"上下文: {context_info}\n\n请简短回复: {last_user_msg.content}"),
    ])

    return {
        "messages": [response],
        "step_count": 1,
    }

def counter(state: State) -> dict:
    """计数节点：记录处理步骤"""
    print(f"  [计数器] 已执行 {state.get('step_count', 0)} 步")
    return {"step_count": 1}

# 构建图：START → analyzer → responder → counter → END
graph = StateGraph(State)
graph.add_node("analyzer", analyzer)
graph.add_node("responder", responder)
graph.add_node("counter", counter)

graph.add_edge(START, "analyzer")
graph.add_edge("analyzer", "responder")
graph.add_edge("responder", "counter")
graph.add_edge("counter", END)

app = graph.compile()
```

三个节点，线性连接。现在执行看看：

```python
result = app.invoke({
    "messages": [HumanMessage(content="Python 的列表推导式怎么用？")],
    "context": "",
    "step_count": 0,
})

print(f"step_count: {result['step_count']}")
print(f"context: {result['context']}")
print(f"消息数量: {len(result['messages'])}")
```

输出：

```
  [计数器] 已执行 2 步
step_count: 3
context: 主题: Python 列表推导式的用法
消息数量: 4
```

逐个拆解这些数字：

**step_count 为什么是 3？** 初始值是 0，analyzer 返回 1（累加到 1），responder 返回 1（累加到 2），counter 返回 1（累加到 3）。

**消息数量为什么是 4？** 初始 1 条 HumanMessage，analyzer 追加了 1 条 AIMessage，responder 追加了 1 条 AIMessage。counter 没有返回 messages 所以不追加。总共 3 条——但 counter 函数里 `state.get('step_count', 0)` 读到的是 responder 执行后的值 2，打印时显示 2。

**context 为什么是 analyzer 设置的值？** analyzer 把 context 设为 `"主题: Python 列表推导式的用法"`，responder 和 counter 都没有返回 context，所以它保持 analyzer 设的值不变。

## 状态在各节点间的变化过程

为了看清 State 在每个节点执行后的变化，可以用 `stream()` 逐步观察：

```python
print("逐步执行过程:")
for event in app.stream({
    "messages": [HumanMessage(content="什么是递归？")],
    "context": "",
    "step_count": 0,
}):
    for node_name, node_output in event.items():
        print(f"  [{node_name}]")
        if "context" in node_output:
            print(f"    context: {node_output.get('context', '(未修改)')}")
        print(f"    step_count 增量: {node_output.get('step_count', 0)}")
        if "messages" in node_output:
            last_msg = node_output["messages"][-1]
            preview = last_msg.content[:60]
            print(f"    新消息: {preview}...")
```

`stream()` 和 `invoke()` 的区别是：`invoke()` 等所有节点执行完返回最终 State；`stream()` 每个节点执行完就返回一次，输出格式是 `{节点名: 该节点的返回值}`。

输出大致如下：

```
逐步执行过程:
  [analyzer]
    context: 主题: 递归的概念
    step_count 增量: 1
    新消息: [分析] 用户在讨论: 递归的概念...
  [responder]
    step_count 增量: 1
    新消息: 递归是指函数调用自身的编程技巧...
  [counter]
    step_count 增量: 1
```

注意 `event` 中每个节点的输出只包含**该节点返回的字段**，不是完整 State。要获取某个节点执行后的完整 State，需要累加之前所有节点的输出。

## 节点只返回需要修改的字段

这是一个容易混淆的点。再看一次 counter 节点：

```python
def counter(state: State) -> dict:
    print(f"  [计数器] 已执行 {state.get('step_count', 0)} 步")
    return {"step_count": 1}
```

它只返回了 `step_count`，没有返回 `messages` 和 `context`。这意味着：

- `step_count`：通过 `operator.add` 累加 1
- `messages`：不受影响，保持之前的消息列表
- `context`：不受影响，保持 analyzer 设置的值

节点不需要返回所有字段，只需要返回它想修改的字段。未返回的字段保持原值不变。这是 LangGraph 的核心设计：每个节点只关心自己需要的数据，返回自己负责的更新，State 框架自动合并。

## 连续调用：复用消息历史

每次 `invoke` 是独立的，图不会自动记住上一次的对话。如果想接着上一次的聊天继续，需要手动把之前的消息传进去：

```python
# 第一次调用
result1 = app.invoke({
    "messages": [HumanMessage(content="什么是递归？")],
    "context": "",
    "step_count": 0,
})

# 第二次调用，把第一次的对话历史带上，再追加新的用户消息
result2 = app.invoke({
    "messages": result1["messages"] + [HumanMessage(content="能举个例子吗？")],
    "context": "",
    "step_count": 0,
})
```

关键是 `result1["messages"] + [HumanMessage(...)]`——把上一次完整的消息历史拼上新问题，LLM 就能看到之前的对话上下文。后面学到 Checkpointer 时会看到更优雅的做法，不需要手动拼接。

## 小结

- **多字段 State** 用 TypedDict 定义，每个字段可以有不同的 reducer
- **add_messages reducer**：新消息追加到列表（相同 id 替换）
- **operator.add reducer**：数值累加（int）或列表拼接（list）
- **无 reducer 的字段**：节点返回的值直接覆盖旧值
- **节点函数** 接收 State，只返回需要修改的字段，未返回的字段保持不变
- **stream()** 逐步返回每个节点的输出，适合观察 State 的中间变化

## 完整代码

把下面代码保存为 `step2_state_management.py`，并和前面的 `config.py` 放在同一个目录下运行：

```python
"""
LangGraph 教程 - Step 2: 状态管理
=================================
演示如何在 StateGraph 中使用复杂状态：
- 多字段的 TypedDict 状态
- reducer 函数（add_messages、operator.add）
- 状态在节点之间的传递和累积
"""

import operator
from typing import Annotated
from typing_extensions import TypedDict

from langchain_core.messages import HumanMessage, AIMessage
from langchain_deepseek import ChatDeepSeek
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages

from config import DEEPSEEK_API_KEY, DEEPSEEK_MODEL, DEEPSEEK_BASE_URL

# ============================================================
# 1. 定义复杂状态
# ============================================================
# State 可以包含多个字段，每个字段可以有不同的 reducer
# - messages: 使用 add_messages reducer，新消息追加到列表
# - context: 普通字段，新值覆盖旧值
# - step_count: 使用 operator.add reducer，新值与旧值相加（累加器）
class State(TypedDict):
    messages: Annotated[list, add_messages]
    context: str
    step_count: Annotated[int, operator.add]

# ============================================================
# 2. 初始化 LLM
# ============================================================
llm = ChatDeepSeek(
    model=DEEPSEEK_MODEL,
    api_key=DEEPSEEK_API_KEY,
    base_url=DEEPSEEK_BASE_URL,
)

# ============================================================
# 3. 定义节点函数
# ============================================================
def analyzer(state: State) -> dict:
    """分析节点：分析用户输入，提取上下文信息"""
    last_message = state["messages"][-1]
    user_input = last_message.content if hasattr(last_message, "content") else str(last_message)

    # 使用 LLM 分析用户意图，生成上下文摘要
    analysis_prompt = f"请用一句话概括以下用户输入的主题：\n\n{user_input}"
    analysis = llm.invoke([HumanMessage(content=analysis_prompt)])

    # 返回需要更新的字段
    # context 是普通字段，新值覆盖旧值
    # step_count 使用 operator.add reducer，返回 1 表示累加 1
    return {
        "context": f"主题: {analysis.content}",
        "messages": [AIMessage(content=f"[分析结果] 用户在讨论: {analysis.content}")],
        "step_count": 1,
    }

def responder(state: State) -> dict:
    """回复节点：根据上下文和消息历史生成回复"""
    # 构造包含上下文的提示
    context_info = state.get("context", "无上下文")
    messages = state["messages"]

    # 把上下文信息注入到系统消息中
    system_msg = HumanMessage(
        content=f"当前对话上下文: {context_info}\n\n"
        f"请基于以上上下文，简短回复用户。"
    )

    # 使用最后一条用户消息作为主要输入
    user_msgs = [m for m in messages if isinstance(m, HumanMessage)]
    last_user_msg = user_msgs[-1] if user_msgs else messages[0]

    response = llm.invoke([system_msg, last_user_msg])

    return {
        "messages": [response],
        "step_count": 1,
    }

def counter(state: State) -> dict:
    """计数节点：记录处理步骤"""
    current_count = state.get("step_count", 0)
    print(f"  [计数器] 当前已执行 {current_count} 个步骤")

    # 只累加 1，不需要返回其他字段
    return {"step_count": 1}

# ============================================================
# 4. 构建图
# ============================================================
# 线性流程: START -> analyzer -> responder -> counter -> END
graph = StateGraph(State)
graph.add_node("analyzer", analyzer)
graph.add_node("responder", responder)
graph.add_node("counter", counter)

graph.add_edge(START, "analyzer")
graph.add_edge("analyzer", "responder")
graph.add_edge("responder", "counter")
graph.add_edge("counter", END)

app = graph.compile()

# ============================================================
# 5. 演示
# ============================================================
def demo_single_invoke():
    """Demo 1: 单次调用，观察状态在各节点间的传递"""
    print("=" * 50)
    print("Demo 1: 单次调用 - 状态在节点间传递")
    print("=" * 50)

    initial_state = {
        "messages": [HumanMessage(content="Python 的列表推导式怎么用？")],
        "context": "",
        "step_count": 0,
    }

    result = app.invoke(initial_state)

    print(f"\n最终状态:")
    print(f"  step_count: {result['step_count']} (经过了 {result['step_count']} 个节点)")
    print(f"  context: {result['context']}")
    print(f"  消息数量: {len(result['messages'])}")

    print(f"\n消息列表:")
    for i, msg in enumerate(result["messages"]):
        role = type(msg).__name__
        content_preview = msg.content[:80] + "..." if len(msg.content) > 80 else msg.content
        print(f"  [{i}] {role}: {content_preview}")
    print()

def demo_accumulated_state():
    """Demo 2: 连续调用，复用消息历史"""
    print("=" * 50)
    print("Demo 2: 连续调用 - 复用消息历史")
    print("=" * 50)

    # 第一次调用
    print("--- 第一次调用 ---")
    result1 = app.invoke({
        "messages": [HumanMessage(content="什么是递归？")],
        "context": "",
        "step_count": 0,
    })
    print(f"  消息数量: {len(result1['messages'])}")

    # 第二次调用，把第一次的对话历史带上，再追加新的用户消息
    print("\n--- 第二次调用（携带历史） ---")
    result2 = app.invoke({
        "messages": result1["messages"] + [HumanMessage(content="能举个例子吗？")],
        "context": "",
        "step_count": 0,
    })
    print(f"  消息数量: {len(result2['messages'])}")

    print(f"\n第二次调用的完整消息历史:")
    for i, msg in enumerate(result2["messages"]):
        role = type(msg).__name__
        content_preview = msg.content[:80] + "..." if len(msg.content) > 80 else msg.content
        print(f"  [{i}] {role}: {content_preview}")
    print()

def demo_state_schema():
    """Demo 3: 查看状态的结构定义"""
    print("=" * 50)
    print("Demo 3: 状态结构（State Schema）")
    print("=" * 50)

    # 查看图的 channels（即状态的各个字段及其 reducer）
    print("状态字段:")
    print(f"  messages: list, reducer=add_messages (追加消息)")
    print(f"  context:  str, 无 reducer (直接覆盖)")
    print(f"  step_count: int, reducer=operator.add (累加)")

    # 也可以从编译后的图获取节点信息
    graph_data = app.get_graph()
    print(f"\n执行路径:")
    for edge in graph_data.edges:
        print(f"  {edge.source} -> {edge.target}")
    print()

if __name__ == "__main__":
    demo_single_invoke()
    demo_accumulated_state()
    demo_state_schema()
```
