前面的代码有个明显问题：程序一关，对话就没了。而且对话轮次多了之后，messages 列表越来越长，最终会撑爆上下文窗口。这篇解决这两个问题。

## 两种记忆

Agent 的记忆分两种：

**短期记忆**（Short-term Memory）：就是对话历史。用户说了什么、AI 回了什么、调了什么工具、结果是什么，这些都存在 messages 列表里，每次请求都发给 LLM，短期记忆的容量受上下文窗口限制。

**长期记忆**（Long-term Memory）：跨对话持久保存的信息。比如用户偏好（"我喜欢简洁的回答"）、项目背景（"这是一个 Spring Boot 项目"）、之前处理过的关键事实（"用户叫张三，邮箱是 xxx"）。长期记忆存在外部存储（文件、数据库）里，需要时检索出来，塞进 messages 作为上下文。

```
短期记忆（messages 列表）：
  用户: 帮我查北京天气
  AI: [调用 get_weather("北京")] 北京晴天 18-28°C
  用户: 那上海呢
  AI: [调用 get_weather("上海")] 上海多云 22-30°C

长期记忆（外部存储）：
  - 用户偏好: 喜欢简洁回答，不需要过多解释
  - 用户信息: 张三，后端开发，主要用 Java
  - 项目背景: Spring Boot 3.2 + Java 17
```

## 短期记忆管理

### 方案一：固定窗口（最简单）

只保留最近 N 轮对话，更早的丢弃：

```python
MAX_HISTORY_ROUNDS = 10  # 保留最近 10 轮

def trim_messages(messages: list, max_rounds: int) -> list:
    """保留 system 消息 + 最近 N 轮对话"""
    system_messages = [m for m in messages if m["role"] == "system"]
    conversation = [m for m in messages if m["role"] != "system"]

    # 每轮 = 1 条 user + 1 条 assistant（可能还有 tool 消息）
    # 按 user 消息分段
    rounds = []
    current_round = []
    for msg in conversation:
        if msg["role"] == "user" and current_round:
            rounds.append(current_round)
            current_round = []
        current_round.append(msg)
    if current_round:
        rounds.append(current_round)

    # 只保留最近 N 轮
    trimmed = rounds[-max_rounds:]
    flat = [msg for round_msgs in trimmed for msg in round_msgs]

    return system_messages + flat
```

使用：

```python
messages = trim_messages(messages, MAX_HISTORY_ROUNDS)
```

优点：简单直接，Token 消耗可控。
缺点：丢弃了早期对话，Agent 丢失了上下文。

### 方案二：Token 预算（更精确）

设定一个 Token 预算，从最早的对话开始丢弃，直到 Token 数在预算内：

```python
import tiktoken

MAX_TOKENS = 4000  # 预算 4000 tokens

def count_tokens(messages: list, model: str) -> int:
    encoding = tiktoken.encoding_for_model(model)
    total = 0
    for msg in messages:
        # 每条消息有固定开销（role 字段等）
        total += 4
        for key, value in msg.items():
            if isinstance(value, str):
                total += len(encoding.encode(value))
            elif isinstance(value, list):
                # tool_calls 等复杂字段
                total += len(encoding.encode(json.dumps(value)))
        total += 2  # 消息分隔符
    return total

def trim_by_token_budget(messages: list, budget: int) -> list:
    system_messages = [m for m in messages if m["role"] == "system"]
    conversation = [m for m in messages if m["role"] != "system"]

    while count_tokens(system_messages + conversation) > budget and conversation:
        conversation.pop(0)  # 从最早的开始丢弃

    return system_messages + conversation
```

比固定窗口更精确，但每次都要计算 Token 数，有一点性能开销。

### 方案三：摘要压缩（不丢信息）

不直接丢弃旧消息，而是让 LLM 把旧消息压缩成一段摘要：

```python
def summarize_messages(messages: list, keep_recent: int = 4) -> list:
    """把早期对话压缩成摘要，保留最近几轮"""
    system_messages = [m for m in messages if m["role"] == "system"]
    conversation = [m for m in messages if m["role"] != "system"]

    if len(conversation) <= keep_recent:
        return messages

    # 分割：早期的压缩，最近的保留
    old_messages = conversation[:-keep_recent]
    recent_messages = conversation[-keep_recent:]

    # 把早期对话发给 LLM 生成摘要
    summary_prompt = "请将以下对话历史压缩成一段简洁的摘要，保留关键信息（用户需求、已完成的操作、重要结论）：\n\n"
    for msg in old_messages:
        role = msg["role"]
        content = msg.get("content", "")
        if content:
            summary_prompt += f"{role}: {content}\n"

    summary_response = client.chat.completions.create(
        model="deepseek-v4-flash",
        messages=[{"role": "user", "content": summary_prompt}],
        temperature=0
    )

    summary = summary_response.choices[0].message.content

    # 用摘要替代早期对话
    summary_message = {"role": "system", "content": f"[之前的对话摘要]\n{summary}"}

    return system_messages + [summary_message] + recent_messages
```

这样旧信息没有完全丢失，而是被压缩了。摘要压缩的缺点是额外调用一次 LLM，有成本和延迟。通常在对话长度达到一定阈值时才触发，而不是每次都压缩。

## 长期记忆

### 简单实现：文件存储

最简单的长期记忆就是存到文件里，用一个 JSON 文件保存记忆条目：

```python
import json
import os

MEMORY_FILE = "memory.json"

def load_memory() -> list:
    if os.path.exists(MEMORY_FILE):
        with open(MEMORY_FILE, "r") as f:
            return json.load(f)
    return []

def save_memory(memories: list):
    with open(MEMORY_FILE, "w") as f:
        json.dump(memories, f, ensure_ascii=False, indent=2)

def add_memory(fact: str):
    memories = load_memory()
    memories.append({"content": fact})
    save_memory(memories)
```

使用时，把长期记忆塞进 system prompt：

```python
memories = load_memory()
memory_text = "\n".join([f"- {m['content']}" for m in memories])

system_prompt = f"""你是一个智能助手。

以下是你记住的关于用户的信息：
{memory_text}

回答时要参考这些信息。如果用户告诉你新的个人信息或偏好，记住它们。"""

messages[0] = {"role": "system", "content": system_prompt}
```

但这种方式有一个问题：记忆条目多了之后，全塞进 system prompt 会占太多 Token，需要一种**检索**机制——每次只把跟当前问题相关的记忆取出来。

### 进阶实现：向量检索

简单文件存储的记忆方案在条目少的时候够用，但条目一多就力不从心——假设你存了 500 条记忆，全塞进 system prompt 可能要消耗几千 Token，而且大部分跟当前问题毫无关系。我们需要一种机制：**根据用户当前的问题，自动找到最相关的几条记忆**，这就是向量检索

#### 什么是向量

向量就是一个有序的数字列表。比如 `[0.2, -0.5, 0.8]` 是一个 3 维向量，它有 3 个数字，每个数字对应一个维度。在 AI 领域，向量通常有几百到几千个维度，比如 `[0.012, -0.034, 0.078, ..., 0.056]` 可能有 1536 个数字

向量本身只是一组数字，没有直观含义。它的价值在于：通过特定的方式生成，可以让向量之间的数学关系反映语义关系

#### 什么是向量模型（Embedding Model）

向量模型（也叫 Embedding 模型）的作用是把一段文本转换成一个向量。你输入一句话，它输出一个固定长度的数字列表

```
"我喜欢吃苹果"  →  [0.012, -0.034, 0.078, ..., 0.056]  (1024维)
"我爱吃水果"    →  [0.015, -0.031, 0.072, ..., 0.049]  (1024维)
"今天天气很好"  →  [0.233, 0.187, -0.042, ..., 0.111]  (1024维)
```

Embedding 模型经过大规模语料训练，学到了一个关键能力：**语义相近的文本，转换出的向量在数值上也接近**。可以理解为：每个向量对应空间中的一个坐标点，数值接近意味着两个点在空间中靠得近。上面"我喜欢吃苹果"和"我爱吃水果"意思相近，所以它们在空间中的点靠得很近；而"今天天气很好"语义不同，对应的点就离得远

阿里云提供的 Embedding 模型 `text-embedding-v4`（输出 1024 维向量），其他厂商如 OpenAI 也有各自的 Embedding 模型。调用方式和 Chat Completion API 类似，只是接口路径不同：

```python
response = embedding_client.embeddings.create(
    model="text-embedding-v4",
    input="我喜欢吃苹果"
)
embedding = response.data[0].embedding  # 一个 1024 维的浮点数列表
```

#### 什么是向量相似度匹配

有了向量之后，还需要一种方法来衡量两个向量"有多接近"，最常用的方法是**余弦相似度**（Cosine Similarity）。

计算公式：两个向量的点积除以它们模长的乘积。用代码实现：

```python
import numpy as np

def cosine_similarity(a: list, b: list) -> float:
    a = np.array(a)
    b = np.array(b)
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# 示例
vec1 = [1.0, 0.0, 0.0]
vec2 = [0.9, 0.1, 0.0]
vec3 = [0.0, 0.0, 1.0]

print(cosine_similarity(vec1, vec2))  # 0.995，非常相似
print(cosine_similarity(vec1, vec3))  # 0.0，完全不相关
```

`vec1` 和 `vec2` 方向接近，余弦相似度接近 1；`vec1` 和 `vec3` 方向垂直，余弦相似度为 0

#### 向量检索的完整流程

把上面的知识串起来，向量检索的流程就是：

1. **存入阶段**：把每条记忆文本通过 Embedding 模型转成向量，存到数据库中
2. **检索阶段**：把用户的提问也转成向量，然后跟所有存储的向量计算余弦相似度，取相似度最高的几条

```
存入阶段：
  "用户叫张三，Java后端开发"  →  Embedding 模型  →  向量 A [0.12, -0.03, ...]
  "项目用 Spring Boot 3.2"   →  Embedding 模型  →  向量 B [0.45, 0.21, ...]
  "用户喜欢简洁的回答"        →  Embedding 模型  →  向量 C [0.08, 0.33, ...]

检索阶段（用户问："帮我写一个 REST 接口"）：
  "帮我写一个 REST 接口"      →  Embedding 模型  →  向量 Q [0.43, 0.19, ...]

  相似度计算：
    Q vs A: 0.45  （张三是Java开发——有一定相关性）
    Q vs B: 0.92  （Spring Boot——高度相关，写REST接口需要框架信息）
    Q vs C: 0.21  （偏好简洁——相关性低）

  → 取 top 2：B 和 A，作为上下文塞进 system prompt
```

#### 代码实现

```python
from openai import OpenAI
import numpy as np

embedding_client = OpenAI(
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
    api_key="your-api-key"  # 阿里云 API Key
)

def get_embedding(text: str) -> list:
    response = embedding_client.embeddings.create(
        model="text-embedding-v4",
        input=text
    )
    return response.data[0].embedding

def cosine_similarity(a: list, b: list) -> float:
    a = np.array(a)
    b = np.array(b)
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

def search_memory(query: str, memories: list, top_k: int = 3) -> list:
    """从记忆中检索跟 query 最相关的 top_k 条"""
    if not memories:
        return []

    query_embedding = get_embedding(query)

    scored = []
    for mem in memories:
        if "embedding" not in mem:
            mem["embedding"] = get_embedding(mem["content"])
        score = cosine_similarity(query_embedding, mem["embedding"])
        scored.append((score, mem))

    scored.sort(key=lambda x: x[0], reverse=True)
    return [mem for _, mem in scored[:top_k]]
```

检索到相关记忆后，只把这些塞进 system prompt：

```python
relevant_memories = search_memory(user_input, all_memories, top_k=3)
memory_text = "\n".join([f"- {m['content']}" for m in relevant_memories])

system_prompt = f"""你是一个智能助手。

相关的背景信息：
{memory_text}
"""
```

向量检索需要额外的 Embedding API 调用，但好处是记忆量大时依然精准——不管你存了 100 条还是 10000 条记忆，每次只取最相关的几条。

实际项目中，向量存储通常用专门的向量数据库（如 Chroma、Pinecone、Milvus）。

## 完整代码（短期记忆 + 长期记忆 + 自动记忆）

        user_input = input("你: ").strip()
        if not user_input or user_input.lower() in ("exit", "quit"):
            break

        # 1. 加载长期记忆，检索相关条目
        all_memories = load_memory()
        relevant = search_memory(user_input, all_memories, top_k=3)
        memory_text = "\n".join([f"- {m['content']}" for m in relevant])

        # 2. 更新 system prompt（带上相关记忆）
        if memory_text:
            messages[0] = {
                "role": "system",
                "content": f"{SYSTEM_PROMPT}\n\n相关记忆:\n{memory_text}"
            }

        # 3. 添加用户消息
        messages.append({"role": "user", "content": user_input})

        # 4. 短期记忆管理：超出预算就压缩
        if count_tokens(messages) > MAX_TOKENS:
            messages = summarize_messages(messages)

        # 5. 执行 Agent 循环
        while True:
            response = client.chat.completions.create(
                model=MODEL,
                messages=messages,
                tools=[WEATHER_TOOL] + MEMORY_TOOLS
            )

            choice = response.choices[0]

            if choice.finish_reason == "stop":
                messages.append({"role": "assistant", "content": choice.message.content})
                print(f"AI: {choice.message.content}\n")
                break

            if choice.finish_reason == "tool_calls":
                messages.append(choice.message.model_dump())

                for tool_call in choice.message.tool_calls:
                    func_name = tool_call.function.name
                    func_args = json.loads(tool_call.function.arguments)

                    result = tool_map[func_name](**func_args)
                    print(f"  [调用工具: {func_name}({func_args})]")
                    print(f"  [工具结果: {result}]")

                    messages.append({
                        "role": "tool",
                        "tool_call_id": tool_call.id,
                        "content": result
                    })


if __name__ == "__main__":
    run()
