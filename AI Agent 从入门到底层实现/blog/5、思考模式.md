前几篇的 Agent 有个问题：它"不思考就行动"。用户问一个复杂问题，LLM 直接给答案，中间没有推理过程。对于简单问题这没问题，但复杂问题容易出错——就像一个人不经思考就脱口而出，说错的可能性很大。

这篇给 Agent 加上"思考"的能力。

## 为什么需要思考模式

看一个例子。用户说：

```
一个商店周一到周五每天卖 100 个苹果，周六卖 200 个，周日卖 150 个。
如果每个苹果成本 2 元，售价 5 元，一周的利润是多少？
```

**没有思考模式**，LLM 可能直接给出一个数字——算对算错全凭运气。

**有思考模式**，LLM 会先展开推理过程，再给出最终答案。中间步骤可见，错了也能发现。

## 模型原生思考模式

现在的模型（如 DeepSeek-V4、Claude Opus 4.7）都内置了思考能力。不需要在 prompt 里教模型"请一步步思考"——模型本身就能先推理再回答。你只需要通过 API 参数告诉模型"请展示你的推理过程"。

以 DeepSeek 为例，通过 `thinking` 参数控制：

```python
response = client.chat.completions.create(
    model="deepseek-v4-flash",
    messages=messages,
    reasoning_effort="high",
    extra_body={"thinking": {"type": "enabled"}}
)
```

| 参数 | 作用 | 可选值 |
|------|------|--------|
| `thinking.type` | 开关思考模式 | `"enabled"`（开启）/ `"disabled"`（关闭） |
| `reasoning_effort` | 推理深度 | `"high"`（默认）/ `"max"`（更深推理） |

开启后，模型会先输出推理过程（Chain of Thought），再输出最终答案。推理过程通过 `reasoning_content` 字段返回，和 `content` 同级：

```python
reasoning = response.choices[0].message.reasoning_content  # 推理过程
content = response.choices[0].message.content               # 最终答案
```

注意：开启思考模式后，`temperature`、`top_p` 等参数无效。模型会自动选择最佳的生成策略。具体详情看：https://api-docs.deepseek.com/zh-cn/guides/thinking_mode


## 实际效果对比

用一个经典的陷阱题来测试：

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://api.deepseek.com",
    api_key="your-api-key"
)

# 关闭思考模式
response = client.chat.completions.create(
    model="deepseek-v4-flash",
    messages=[{"role": "user", "content": "9.11 和 9.8 哪个大？"}],
    extra_body={"thinking": {"type": "disabled"}}
)
print(response.choices[0].message.content)
```

关闭思考模式的输出：

```
9.11 更大。因为 11 > 8。
```

错了！9.11 的十分位是 1，9.8 的十分位是 8，所以 9.8 > 9.11。但因为 11 > 8 的直觉太强，模型直接给出了错误答案。

再试开启思考模式：

```python
# 开启思考模式
response = client.chat.completions.create(
    model="deepseek-v4-flash",
    messages=[{"role": "user", "content": "9.11 和 9.8 哪个大？"}],
    reasoning_effort="high",
    extra_body={"thinking": {"type": "enabled"}}
)

msg = response.choices[0].message
print(f"[推理过程]\n{msg.reasoning_content}\n")
print(f"[最终答案]\n{msg.content}")
```

开启思考模式的输出：

```
[推理过程]
这个问题很简单，9.11 和 9.8 比较大小，9.8 更大。因为 9.8 = 9.80，而 9.11 < 9.80。不需要调用工具。

[最终答案]
9.8 更大。

比较方法：9.8 = 9.80，而 9.11 < 9.80，所以 **9.8 > 9.11**。
```

模型在推理过程中把 9.8 补齐为 9.80，然后逐位比较，得出了正确答案。这就是思考模式的价值——让模型"想清楚再回答"。

## 在 Agent 中集成思考模式

把思考模式集成到 Agent 的执行循环里，只需要做两处改动：

```python
def run_agent(user_input: str, messages: list) -> str:
    messages.append({"role": "user", "content": user_input})

    while True:
        response = client.chat.completions.create(
            model="deepseek-v4-flash",
            messages=messages,
            tools=tools,
            reasoning_effort="high",
            extra_body={"thinking": {"type": "enabled"}}
        )

        choice = response.choices[0]
        msg = choice.message

        # 打印推理过程
        reasoning = getattr(msg, "reasoning_content", None)
        if reasoning:
            print(f"[思考过程]\n{reasoning}\n")

        if choice.finish_reason == "stop":
            messages.append(msg)
            return msg.content

        if choice.finish_reason == "tool_calls":
            messages.append(msg)

            for tool_call in msg.tool_calls:
                func_name = tool_call.function.name
                func_args = json.loads(tool_call.function.arguments)

                print(f"  [调用工具: {func_name}({func_args})]")
                result = tool_map[func_name](**func_args)
                print(f"  [工具结果: {result}]")

                messages.append({
                    "role": "tool",
                    "tool_call_id": tool_call.id,
                    "content": result
                })
```

相比不带思考的版本，只多了 `extra_body` 和 `reasoning_effort` 两个参数，以及打印 `reasoning_content` 的逻辑。

运行效果：

```
你: 北京天气怎么样？适合出门跑步吗？
[思考过程]
用户想知道北京的天气情况，以及是否适合出门跑步。我需要查询北京的天气信息。让我调用get_weather工具来获取天气数据。

  [调用工具: get_weather({'city': '北京'})]
  [工具结果: 晴，18-28°C]
[思考过程]
天气是晴天，温度在18-28°C之间。这是一个比较舒适的温度范围，适合跑步。让我给用户一个简洁的建议。

AI: 北京今天 **晴**，气温 **18~28°C**，非常适合出门跑步！
建议：
- **最佳时间**：早晨或傍晚（避开中午高温）
- **注意**：紫外线较强，记得防晒补水
```

可以看到，模型在调用工具前先推理出"我需要查天气"，拿到结果后又推理出"温度适合跑步"。思考过程让 Agent 的决策更透明、更可信赖。

## 带工具调用时的注意事项

当思考模式与 Function Calling 一起使用时，有一个关键点：如果模型在某一轮执行了工具调用，那这一轮的 `reasoning_content` **必须**回传给 API。不能丢弃。

```python
# 正确：直接 append 整个 message 对象
messages.append(response.choices[0].message)

# 这等价于：
messages.append({
    "role": "assistant",
    "content": msg.content,
    "reasoning_content": msg.reasoning_content,  # 工具调用轮次必须保留
    "tool_calls": msg.tool_calls
})
```

如果丢弃了工具调用轮次的 `reasoning_content`，API 会返回 400 错误。

而如果模型没有执行工具调用（`finish_reason == "stop"`），`reasoning_content` 可以不回传——API 会忽略它。

## reasoning_effort 参数

`reasoning_effort` 控制推理的深度：

| 值 | 效果 | 适用场景 |
|----|------|----------|
| `"high"` | 标准推理深度（默认） | 大多数场景 |
| `"max"` | 最深推理 | 复杂数学、逻辑推理、代码分析 |

`"max"` 会消耗更多 Token 和时间，只在真正需要深度推理时使用。

## 完整代码

把前面的思考模式集成到 Agent 执行循环中，得到如下完整代码：

```python
# 运行方式：pip install openai
# 将 your-api-key 替换为你的 DeepSeek API Key，保存为 thinking_agent.py 后运行：python thinking_agent.py

import json
from openai import OpenAI

# 创建客户端
client = OpenAI(
    base_url="https://api.deepseek.com",
    api_key="your-api-key"  # 替换为你的 DeepSeek API Key
)

# 模拟天气查询工具
def get_weather(city: str) -> str:
    """查询指定城市的天气信息"""
    mock_data = {
        "北京": "晴，18-28°C",
        "上海": "多云，20-26°C",
        "深圳": "阵雨，25-32°C",
        "成都": "阴，16-22°C",
    }
    return mock_data.get(city, f"{city}：暂无天气数据")

# 工具定义（JSON Schema 格式，告诉模型有哪些工具可用）
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "查询指定城市的天气信息，包括天气状况和温度范围",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "要查询天气的城市名称，例如：北京、上海、深圳"
                    }
                },
                "required": ["city"]
            }
        }
    }
]

# 工具名 -> 工具函数的映射
tool_map = {
    "get_weather": get_weather
}

SYSTEM_PROMPT = """你是一个智能助手，可以根据用户的问题调用合适的工具来获取信息，然后给出回答。
回答时要基于工具返回的实际数据，不要编造信息。"""

def run_agent(user_input: str, messages: list) -> str:
    """执行一轮 Agent 对话，返回最终回复文本"""
    messages.append({"role": "user", "content": user_input})

    while True:
        # 开启思考模式：reasoning_effort="high" + thinking enabled
        response = client.chat.completions.create(
            model="deepseek-v4-flash",
            messages=messages,
            tools=tools,
            reasoning_effort="high",
            extra_body={"thinking": {"type": "enabled"}}
        )

        choice = response.choices[0]
        msg = choice.message

        # 打印推理过程
        reasoning = getattr(msg, "reasoning_content", None)
        if reasoning:
            print(f"[思考过程]\n{reasoning}\n")

        # 模型直接给出最终回答
        if choice.finish_reason == "stop":
            # 直接 append 整个 msg 对象，reasoning_content 会一同保留
            messages.append(msg)
            return msg.content

        # 模型决定调用工具
        if choice.finish_reason == "tool_calls":
            # 关键：必须 append 整个 msg，工具调用轮次的 reasoning_content 不能丢弃
            # 否则 API 会返回 400 错误
            messages.append(msg)

            for tool_call in msg.tool_calls:
                func_name = tool_call.function.name
                func_args = json.loads(tool_call.function.arguments)

                print(f"  [调用工具: {func_name}({func_args})]")
                result = tool_map[func_name](**func_args)
                print(f"  [工具结果: {result}]")

                messages.append({
                    "role": "tool",
                    "tool_call_id": tool_call.id,
                    "content": str(result)
                })

            # 循环继续，把工具结果再次发给模型，模型会继续推理

if __name__ == "__main__":
    messages = [{"role": "system", "content": SYSTEM_PROMPT}]

    print("=== 带思考模式的 Agent（输入 exit 退出）===\n")

    while True:
        user_input = input("你: ")
        if user_input.strip().lower() == "exit":
            print("再见！")
            break

        reply = run_agent(user_input, messages)
        print(f"AI: {reply}\n")
```
